# **DETAILED CRASH ANALYSIS REPORT: Auto Trades & Combo Trades**

## **Executive Summary**

Your Auto Trades and Combo Trades pages crash when switching tabs or when data streams stop/resume. After analyzing both components (2,300+ lines total), I've identified **5 critical root causes** and **3 secondary issues** causing system crashes. All stem from **memory management, resource cleanup, and async state updates**.

## **ROOT CAUSE ANALYSIS**

### **🔴 CAUSE #1: WebSocket Subscriptions NOT Properly Cleaned Up on Tab Switch**

**Severity**: 🔴 CRITICAL  
**Location**: Both Auto Trades & Combo (`useEffect` cleanup missing)

#### The Problem:

When you switch tabs away from Auto Trades/Combo, the component unmounts but **WebSocket subscriptions continue running in the background**:

```tsx
// ❌ BROKEN - Combo.tsx line 782-809
useEffect(() => {
    if (!show_combo) {
        if (runningRef.current) {
            runningRef.current = false;
            setIsRunning(false);
            try {
                run_panel.setIsRunning(false);
            } catch {}
        }
        unsubscribeAll();  // ✅ This is called, BUT...
        return;
    }
    // ... subscribe logic
}, [show_combo, subscribeAll, unsubscribeAll, run_panel]);
```

**The Issue**: 
- `unsubscribeAll()` is called when `show_combo` becomes false
- BUT the subscriptions still have **reference callbacks** (`handleTick`, `handleCandle`) that try to update state
- When new data arrives, the callback fires → `setError`, `setTotalPnl` → **state update on unmounted component** → **React warning + memory leak**
- When you switch back, old subscriptions are still active → data gets duplicated to both old and new subscriptions

#### Proof in Code:

**Auto Trades** - Line 1538-1565:
```tsx
useEffect(() => {
    if (show_auto) {
        if (api_base.api) {
            startSubscriptions();  // ✅ Starts subs
        }
    } else {
        if (runningRef.current) {
            runningRef.current = false;
            setIsRunning(false);
        }
        stopSubscriptions();  // ✅ Stops subs
    }
    return undefined;  // ⚠️ No cleanup return - doesn't clean on unmount
}, [show_auto, run_panel, startSubscriptions, stopSubscriptions]);
```

**Combo** - Line 811-820:
```tsx
useEffect(
    () => () => {
        stopTrading();
        try { run_panel.setIsRunning(false); } catch {}
        unsubscribeAll();  // ✅ This runs on unmount
    },
    [unsubscribeAll, run_panel, stopTrading]  // ⚠️ Stale dependency - could retrigger
);
```

### **🔴 CAUSE #2: Race Condition - State Updates After Unmount**

**Severity**: 🔴 CRITICAL  
**Location**: Auto Trades (line 876-878, 968-976)

#### The Problem:

The component checks `if (!unmountedRef.current)` to avoid state updates after unmount, BUT:

```tsx
// Line 852-888 (Auto Trades)
const pollContractResult = (contractId: number): Promise<Record<string, any>> =>
    new Promise(resolve => {
        const abortController = new AbortController();
        pollCancellationRef.current = abortController;
        const { signal } = abortController;
        const check = async () => {
            if (signal.aborted) {
                resolve({ profit: 0, is_sold: true });
                return;
            }
            try {
                const resp = await (api_base.api as any).send({
                    proposal_open_contract: 1,
                    contract_id: contractId,
                });
                if (signal.aborted) {
                    resolve({ profit: 0, is_sold: true });
                    return;
                }
                const c = resp?.proposal_open_contract;
                if (!c) {
                    setTimeout(check, 800);  // ⚠️ Retries forever, even if unmounted
                    return;
                }
                if (!unmountedRef.current) {
                    pushContract(getContractSnapshot(c));  // ✅ Protected
                }
                if (c.is_sold) {
                    emitContractSoldStatus(c);
                    resolve(c);
                } else setTimeout(check, 800);
            } catch {
                resolve({ profit: 0, is_sold: true });
            }
        };
        check();
    });
```

**The Issue**:
- `setTimeout(check, 800)` continues polling **even after component unmounts**
- If you switch tabs while a contract is being polled, the polling continues
- When component remounts, you may have **2-3 polling loops** running simultaneously
- Eventually all 3 try to call `pushContract()` at same time → race condition → crash

    // Stop if already resolved
